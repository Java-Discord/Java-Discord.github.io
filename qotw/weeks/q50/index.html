  • Submission by Jadefalke#3799

An ArrayList and a LinkedList are both implementations of the List interface but work fundamentally different.

An ArrayList is backed by an Array which grows in size (in a special way, doubling it's size).

A LinkedList is backed using Nodes with a previous and next Node reference for each node.

  • Submission by dan1st#7327

ArrayList and LinkedList are two classes implementing the List representing a list data structure that allows indexed access and adding/removing elements. As the name suggests, ArrayList is based on arrays. An ArrayList instance contains an underlaying array of its elements. When adding an element to the list, it is added to the array. If that would exceed the array size, a new but bigger array is created. On the other hand, LinkedList represents a doubly linked list data structure. It contains a reference to the first and last node. Each node contains a reference to the referenced element, the previous and next node. This allows iterating, inserting and removing elements/nodes. Aside from List, LinkedList also implements the interfaces Queue and Dequeue. While ArrayList has better read more performant than LinkedList and is faster when setting elements at arbitiary indexes or removing and adding (as long as the underlaying array's size is not exceeded) elements at the end, LinkedList is better when adding/removing elements at the begginning as the underlaying array of ArrayList would need to be shifted.

They are both part of the collections framework and implement List<T>. An ArrayList surprisingly works somewhat like a resizable array (hence the name) and a LinkedList uses nodes or something. It also implements Queue and Dequeue.

  • Submission by Gaming32#6509

ArrayLists use an array, while LinkedLists are literally linked lists. Both have advantages and disadvantages for different purposes. ArrayLists are useful for variable-length arrays, while LinkedLists are useful for queues and deques.

ArrayList: Removal/add from/to left end: O(n) Removal/add from/to right end: O(1) Random access: O(1)

LinkedList: Removal/add from/to left end: O(1) Removal/add from/to right end: O(1) Random access: O(n)

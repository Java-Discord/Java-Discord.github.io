Week 40

15.08.2021 - 22.08.2021#

  • Submission by dan1st#7327

The law of demeter states that methods should only access the this object, objects created by it, objects passed to it or objects that are attributes of the this object. For example, the following code would be acceptable according to the law of Demeter:

public class SomeClass{
private String s="Hello World";
public void test(String a){
doSomething(s);
doSomething(a.length);
doSomething(this);
doSomething(toString());
doSomething(s);
}
//doSomething
}

On the other hand, foreign objects should not be accessed.

public class AgainstDemeter{
private SomeComplicatedClass obj;//init
public void someMethod(){
obj.getSomething().getSomethingElse().getYetAnotherThing().doSomething();
}
}

Law of Demeter is the law which states that a single method must not contain all the information. It should be fragmented into sub-methods to divide access to private data.

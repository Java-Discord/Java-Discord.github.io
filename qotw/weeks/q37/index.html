a class that can have only one object, an instance of the class, at a time

  • Submission by Krystilize#7980

A singleton class is a class that has a single instance. Typically these are implemented using a public static final field, with the singleton having a private constructor. However modern approaches sometimes incorporate an enum with a single enum instance. This approach initialises the singleton before standard classes.

  • Submission by dan1st#7327

A singleton is a class where only a single object/instance of a class exists and no other objects can be created. A simple version that uses enums looks like this:

public enum SomeSingleton{
INSTANCE;
//Some code
}

Having just one enum constant makes sure that only one instance (the enum constant named INSTANCE) exists).

For lazy initialization, class loading can be abused in order to create a singleton.

public class SomeSingleton{
private SomeSingleton(){
//Prevent instantiation from outside
}
private static class SomeSingletonHolder{
private static final SomeSingleton INSTANCE=new SomeSingleton();
}
public SomeSingleton getInstance(){
return SomeSingletonHolder.INSTANCE;
}
}

Class loading works on demand so the object is created as soon as the class is used at first. The private constructor makes sure that no other instance is created.

Week 47

10.09.2021 - 24.10.2021#

  • Submission by dan1st#7327

Switch Expressions were introduced in Java 14 and are an improvement for switch-case statements that all assign something to the same value. Instead of:

String someVariable;
switch(something){
case 0:
someVariable="zero"
break;
case 1:
someVariable="one"
break;
default:
someVariable=String.valueOf(someVariable);
}

It is possible to write:

String someVariable=switch(something){
case 0->"zero";
case 1->"one";
default->String.valueOf(something);
}

It also works if branches contain multiple statements using the yield keyword:

String someVariable=switch(something){
case 0->"zero";
case 1->"one";
default:{
System.out.println("other);
yield String.valueOf(something);
}

This syntax allows to write shorter code that is less error-prone as break statements are not required. Switch expressions can be used everywhere where other expressions can be used (variable assignments, method calls, return calls, etc.).

  • Submission by Jadefalke#3799

Before enhanced switch statements you had to write lots of code for a simple switch. You had to break each statement and it wasn't feeling very java like. They looked like this.

switch (value) {
case someValue:
someAction();
break;
case anotherValue:
case someOtherValue:
someOtherAction();
somethingElseToo();
break;
default:
somethingCompletelyElse();
}

Since Java 14 you can use enhanced switch statements which look the following:

switch (value) {
case someValue -> someAction();
case someOtherValue, anotherValue -> {
someOtherAction();
somethingElseToo();
}
default -> somethingCompletelyElse();
}

This is much cleaner and is way easier to read and write.

  • Submission by Maeve#6666

The switch statement has been a part of Java since the beginning. There are some limitations and inconveniences to it tho, one can only have one value per case and fall-through.

Java 12 brought some improvements to it: switch expressions and the possibility to have multiple values per case. To use a switch statement as an expression, the yield keyword was added (Originally, it was break followed by a value, but in Java 13 it was replaced with yield). It is acts similarly to the return keyword but for switch statements. There is no need for a break; after using yield. To avoid fall-through, case -> can be used instead of case:. There is no need for breakusing this syntax. However, it isn't possible to use both syntaxes in the same switch statement/expression.

  • Submission by dan1st#7327

A constructor is called when an object of a certain class is instantiated. Such a constructor can have an arbituary number of parameters and all of these parameters need to be present when instantiating the object. However, it is possible to create multiple constructors with different parameters and the compiler will choose the correct constructor for each object instantiation depending on the argument types:

public class SomeClass{
private final String s;
public SomeClass(){
s="";
}
public SomeClass(String someString){
s=someString;
}
public SomeClass(int someInt){
s=""+someInt;
}
}

Instances of this class could be created like this:

SomeClass empty=new SomeClass();
SomeClass str=new SomeClass("Hello World");
Someclass num=new SomeClass(1337);

Howevery the following is not allowed as no fitting constructor is available:

SomeClass doesNotWork=new SomeClass(13.37);//no constructor with 'double' parameter
  • Submission by Jadefalke#3799

Constructor overloading refers to having multiple constructors for the same class, that have different parameter-lists. What this allows you to do, is to have a base constructor with all possible parameters and overload that constructor with other constructors that have less parameters. If you then call the base-constructor with this you can essentially create optional parameters. An example would be

public SomeClass (int a, String b, List<Integer> c) {
this.a = a;
this.b = b;
//...
}
public SomeClass (int a, List<Integer> c) {
this(a, "default String :)", c)
}
etc ...

This is used almost everywhere in the java API. Having multiple constructors to choose from is a huge advantage and allows you to code much more efficiently and faster!!

  • Submission by Maeve#7872

A constructor is a method to allocate memory for a class instance and initialize the classes attributes (that's probably why it's def __init__() in python (the more you know)). If a class doesn't have a constructor, the Java compiler will use the default constructor and set the fields to null. Constructor overloading is very similar to method overloading. It is used to create objects in a different way and to initialize the member variables in a different way. There is no limit to the amount of overloaded constructors, as long as they have a different amount of args or different types of args.
